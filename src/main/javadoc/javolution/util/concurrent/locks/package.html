Provides interfaces and classes for locking and thread synchronization, offering finer-grained control than standard Java monitor locks, based on the original util.concurrent library by Doug Lea.

Overview:

This package includes the following synchronization utilities:

  • Locking Abstraction: The {@link javolution.util.concurrent.locks.Sync Sync} interface provides a uniform API for lock acquisition (including non-blocking and timed options) and release.
  • Lock Implementations: Mutual exclusion locks such as {@link javolution.util.concurrent.locks.Mutex Mutex} and {@link javolution.util.concurrent.locks.ReentrantLock ReentrantLock}.
  • Read-Write Locks: The {@link javolution.util.concurrent.locks.ReadWriteLock ReadWriteLock} interface and implementations like {@link javolution.util.concurrent.locks.WriterPreferenceReadWriteLock WriterPreferenceReadWriteLock} and {@link javolution.util.concurrent.locks.FIFOReadWriteLock FIFOReadWriteLock} to support multiple concurrent readers and exclusive writers.
  • Condition Variables: {@link javolution.util.concurrent.locks.CondVar CondVar} classes that allow threads to suspend execution until a specific condition is met, functioning similarly to monitor wait/notify but associated with a specific {@link javolution.util.concurrent.locks.Sync Sync} instance.
